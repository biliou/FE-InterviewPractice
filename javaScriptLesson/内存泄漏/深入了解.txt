1.内存泄漏会导致什么？
系统运行缓慢，崩溃，高延时

2.什么是内存泄露？
那些无法被应用所使用但又没有被操作系统回收掉的内存。

3.导致内存泄露的可能因素
	3.1 全局变量引起
		有两种方式的声明变量：
		1. function foo(arg) {
			    bar = "this is a hidden global variable"; //其实相当于window.bar
			}
		2.function foo() {
			    this.variable = "potential accidental global"; //其实也是全局变量
			}
			
	3.2 被遗忘的计时器
		计时器中引用删除的节点，且节点绑定了监听事件。
		解决方法：手动删除监听事件。
		
	3.3 子元素存在引用
		有时候我们会在数据结构中存储 DOM 结点。子节点不删除，父节点也不会被删除。
		解决方法：手动删除DOM节点。document.body.removeChild（）
		
	3.4 闭包
		父函数中有两个闭包共享同一个作用域。其中一个闭包引用了父函数变量并使用，另一个引用了父函数的变量
		但实际上没有使用，导致泄漏。
	------------------------------
		闭包的概念：
		能够读取其他函数内部变量的函数。如f1是f2的外部函数，f2可以将f1中的内部变量返回给f1外部。
		本质上，闭包是将函数内部和函数外部连接起来的桥梁。
		
		使用地方：
		1.让f1内部变量始终保持在内存中
		2.让f1外部可以读取函数内部的变量

4.如何通过chrome观察内存泄漏
	4.1使用Chrome TimeLine观察
	打开memory概述，如果在大收集结束后，使用的内存总量高于开始时
